# Git 子模块管理

## 概述
&emsp;&emsp;在Git仓库中为了避免同一个仓库中的文件内容过于繁杂，可以通过一个聚合工程，使用子模块的方式管理同步其他Git仓库中的项目工程。

## 基本使用

- **添加指定的仓库为当前工作仓库的子模块**
    ``` bash
    # 添加子模块工程
    git submodule add -- git@gitlab.owner.com:remote-a.git remote-a
    # 初始化并更新子模块
    git submodule update --init --recursive -- remote-a
    ```
